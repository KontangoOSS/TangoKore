**Verify SDK ↔ Controller Integration End-to-End**

## Architecture Overview

```
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│  SDK (TangoKore)                Controller (schmutz)           │
│  ────────────────                ──────────────────             │
│                                                                │
│  kontango enroll           GET /install (serve script)         │
│         │                         ↓                            │
│         │                  [Session token + env vars]          │
│         │                         ↓                            │
│         └──→ POST /api/enroll/stream (SSE)                     │
│                  [Minimal fingerprint + session]               │
│                         ↓                                      │
│                   [Server verifies + decides]                  │
│                         ↓                                      │
│              [Stream: verify events + identity]                │
│                         ↓                                      │
│         [Save cert + config, install tunnel]                   │
│                         ↓                                      │
│              ✅ Machine on mesh                               │
│                                                                │
│  ALTERNATIVE: WebSocket Path                                  │
│  ────────────────────────────                                  │
│  kontango enroll           GET /install                        │
│         │                   [Session token]                    │
│         └──→ WS /api/ws/enroll                                 │
│              [Hello → 4 probes → identity]                     │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

## Test Scenario 4: With Pre-Provisioned Credentials (Skip Quarantine)

### Setup

```bash
# Pre-provision AppRole credentials in Bao
# (This would normally be done by an admin)

# Get Bao role ID and secret ID
BAO_ROLE_ID="role-test-machine"
BAO_SECRET_ID="secret-test-machine"

# Store them securely
export KONTANGO_ROLE_ID="$BAO_ROLE_ID"
export KONTANGO_SECRET_ID="$BAO_SECRET_ID"
```

### Execution

```bash
# Download installer WITH credentials
curl -k https://localhost/install \
  -H "X-Role-ID: $BAO_ROLE_ID" \
  -H "X-Secret-ID: $BAO_SECRET_ID" \
  -o install-trusted.sh

chmod +x install-trusted.sh
./install-trusted.sh

# Watch for:
# 1. Fingerprinting
# 2. Enrollment WITH credentials
# 3. Server verifies AppRole
# 4. Decision: "status=trusted" (SKIP quarantine!)
# 5. Full permissions immediately
```

### Verification

```bash
# Check status
kontango status

# Should show:
# Status: trusted
# Permissions: stage-3 (full access)
# No quarantine period!
```

## Test Scenario 6: WebSocket Alternative Path

### Setup

```bash
# Test the WebSocket enrollment endpoint
# (alternative to SSE)

# Create a simple WebSocket client test
cat > ws-test.go << 'EOF'
package main

import (
	"context"
	"fmt"
	"nhooyr.io/websocket"
	"encoding/json"
	"time"
)

func main() {
	ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
	defer cancel()
	
	// Connect to WS endpoint
	conn, _, err := websocket.Dial(ctx, "wss://localhost/api/ws/enroll", nil)
	if err != nil {
		panic(err)
	}
	defer conn.Close(websocket.StatusNormalClosure, "")
	
	// Send hello
	hello := map[string]interface{}{
		"type": "hello",
		"method": "new",
	}
	data, _ := json.Marshal(hello)
	conn.Write(ctx, websocket.MessageText, data)
	
	// Read probes
	for i := 0; i < 4; i++ {
		_, data, _ := conn.Read(ctx)
		fmt.Printf("Probe %d: %s\n", i, string(data))
	}
	
	// Read identity
	_, data, _ := conn.Read(ctx)
	fmt.Printf("Identity: %s\n", string(data))
}
EOF

go run ws-test.go
```

### Verification

```bash
# Check WebSocket probe responses
# Should see 4 probes:
# 1. os
# 2. hardware
# 3. network
# 4. system

# Then identity response with cert + config
```

## Complete End-to-End Flow Diagram

```
USER RUNS INSTALLER
       │
       ├─→ GET /install
       │   ├─ Server fingerprints connection
       │   ├─ Server verifies not banned
       │   ├─ Server generates session token
       │   └─ Returns install script
       │
       ├─→ Script runs on machine
       │   ├─ Collects OS (auto)
       │   ├─ Collects arch (auto)
       │   ├─ Generates/uses machine ID
       │   └─ Collects optional details (hostname, MACs, etc.)
       │
       ├─→ POST /api/enroll/stream
       │   ├─ Sends: OS + arch + ID + optional details
       │   ├─ Includes: session token
       │   ├─ Server receives fingerprint
       │   │
       │   ├─ SERVER DECISION LOGIC:
       │   │  ├─ Has credentials? → TRUSTED path
       │   │  ├─ Fingerprint matches history? → APPROVED path
       │   │  └─ Unknown? → QUARANTINE path
       │   │
       │   ├─ Server runs verification checks:
       │   │  ├─ fingerprint_match: true/false
       │   │  ├─ os_validation: passed/failed
       │   │  ├─ banned_check: passed/failed
       │   │  └─ ...more checks...
       │   │
       │   ├─ Server makes decision:
       │   │  └─ decision: {status: "quarantine|approved|trusted"}
       │   │
       │   └─ Server streams events back:
       │      ├─ verify event (each check)
       │      ├─ decision event (trust level)
       │      └─ identity event (cert + config)
       │
       ├─→ Script receives identity
       │   ├─ Saves certificate to ~/.kontango/
       │   ├─ Saves config (Ziti endpoint, ACL)
       │   ├─ Installs Ziti tunnel
       │   ├─ Starts Ziti tunnel service
       │   ├─ Waits for tunnel to connect
       │   ├─ Installs agent (optional)
       │   ├─ Installs Caddy (optional)
       │   └─ Reports status
       │
       └─→ RESULT:
           ├─ Machine on mesh (Ziti overlay)
           ├─ Authenticated by fingerprint
           ├─ ACLs applied per trust level
           ├─ Agent reporting telemetry (optional)
           └─ ✅ Ready to go!
```

## Validation Checklist

### After Each Enrollment

- [ ] Installer downloaded successfully
- [ ] Fingerprint collected (no errors)
- [ ] Enrollment sent to /api/enroll/stream
- [ ] Verify events received and logged
- [ ] Decision event received
- [ ] Identity (certificate + config) received
- [ ] Certificate saved to ~/.kontango/identity.p12
- [ ] Config saved to ~/.kontango/config.json
- [ ] Ziti tunnel installed
- [ ] Ziti tunnel service started
- [ ] Tunnel connected (ziti edge ls identities shows machine)
- [ ] ACLs applied correctly
- [ ] Status matches expected (quarantine/approved/trusted)

### Cross-Checks

```bash
# SDK side:
kontango status                          # Should show enrolled status
ls -la ~/.kontango/                      # Identity files present
ziti edge ls identities                  # Machine shows in Ziti

# Controller side:
# Check database:
curl -k https://localhost/api/machines   # Should list enrolled machines
# Check enrollment logs for correct decision path
```
